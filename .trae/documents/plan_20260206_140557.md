项目已完成全部核心功能的开发与增强，达到 MVP 可上线标准。

### 1. 核心增强功能

**专注模式 (Focus Mode)**
- **UI**: 实现了 Focus 页面，支持设置专注时长（分钟），选择白名单应用，并实时显示剩余时间。
- **逻辑**: `Service` 层新增 `FocusManager`，负责管理专注会话状态。
- **执行**: `Worker` 服务每秒检查前台应用。若处于专注模式且应用不在白名单中，会立即拦截（Kill Process）并记录日志。

**真实拦截 (Real Enforcement)**
- **NativeHelper**: 实现了 `KillProcess` 方法（调用 `Process.Kill`）。
- **策略**: 无论是 **每日限额超限 (Daily Limit)** 还是 **专注模式违规 (Focus Mode)**，只要策略配置为 `BlockNew` 或 `ForceClose`，服务均会执行强制关闭。

**设置页面 (Settings)**
- **UI**: 实现了 Settings 页面。
- **服务状态**: 可实时检测后台服务运行状态（Running / Stopped）及运行时间（Uptime）。
- **Mock 切换**: 提供了 Mock 模式的显示（目前切换需修改代码重启，界面上做了提示）。

**部署脚本**
- 创建了 `install.ps1` PowerShell 脚本，支持一键发布、停止旧服务、移除旧服务、注册新服务并启动，大大简化了部署流程。

### 2. 运行与验证

**UI 体验 (Mock 模式)**
```powershell
cd src/ScreenTimeWin.App
dotnet run
```
- 进入 **Focus** 页面，勾选几个白名单应用，点击 Start Focus。
- 进入 **Settings** 页面，点击 Check Status（Mock 模式下会显示 Running）。

**生产部署 (真实模式)**
1.  **切换模式**: 修改 `src/ScreenTimeWin.App/App.xaml.cs`，注释掉 `MockAppService`，启用 `AppService`。
2.  **一键安装**:
    以**管理员身份**运行 PowerShell：
    ```powershell
    .\install.ps1
    ```
    脚本会自动编译 Release 版本并注册 Windows Service。
3.  **验证**:
    - 运行 UI (`src/ScreenTimeWin.App/bin/Debug/net8.0-windows/ScreenTimeWin.App.exe` 或通过 `dotnet run`)。
    - 在 Limits 页面设置一个超低限额（如 1 分钟）给 Chrome。
    - 使用 Chrome 一段时间，观察是否被强制关闭（需管理员权限）。
