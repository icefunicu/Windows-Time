功能开发已全部完成，并且已切换为实际使用模式。

### 1. 切换至生产模式
- **App.xaml.cs**: 已禁用 `MockAppService`，启用 `AppService`。现在的 UI 将尝试连接真实的后台服务获取数据。
- **注意**: 由于切换到了真实服务，请确保通过 `install.ps1` 安装并启动了 Windows 服务，否则 UI 将显示连接失败或数据为空。

### 2. 新增数据管理功能
响应“完整可上线产品”的要求，补全了数据管理能力：
- **清除数据 (Clear Data)**:
  - 在 **Settings** 页面新增了“Clear All Data”按钮。
  - 点击确认后，会通过 IPC 通知后台服务清空数据库中的使用记录 (`UsageSessions`, `DailyAggregates`)，但保留配置（规则、白名单）。
- **导出数据 (Export Data)**:
  - 在 **Settings** 页面新增了“Export Data (CSV)”按钮。
  - 后台服务会将历史会话数据导出为 CSV 文件并保存到“我的文档”目录，导出成功后 UI 会弹窗提示文件路径。

### 3. 代码修正与验证
- 修复了 `DataRepository_Extensions.cs` 和 `IAppService.cs` 中的语法错误（缺少括号）。
- 执行了 `dotnet build`，确认所有项目（App, Service, Core, Data, IPC, Tests）均编译通过。

### 4. 上线步骤
现在项目已准备就绪，请按以下步骤上线：
1. **停止旧服务**（如果正在运行）:
   ```powershell
   sc stop ScreenTimeWin
   ```
2. **重新部署**:
   以管理员身份运行 `install.ps1`，脚本会自动编译最新代码并更新服务。
3. **运行客户端**:
   运行 `ScreenTimeWin.App.exe`，即可开始真实的屏幕时间统计与管理。
